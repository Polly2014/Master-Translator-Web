# 📖 Demo 文件使用指南

## 🎯 为什么需要 Demo 专用文件？

### 问题
- ❌ 完整书籍太长（2633行）
- ❌ 翻译时间过长（10-30分钟）
- ❌ Demo 时观众等待焦虑
- ❌ 难以展示实时效果

### 解决方案
✅ **创建精简的 Demo 专用文件**
- 保留书籍结构和风格
- 压缩到 3-10% 的长度
- 翻译时间 1-5 分钟
- 完美展示系统能力

## 🔧 Demo 配置优化

### 方法 1: 使用 Quick Demo 文件（推荐）
```bash
# 不需要修改代码，直接使用精简文件
demo_files/Mustafa_Book_Quick_Demo.md  # 60-90秒
demo_files/Mustafa_Book_Demo.md       # 3-5分钟
```

### 方法 2: 调整分块大小（高级）
如果要演示完整书籍，可以临时调小分块：

```python
# app.py, line ~77
CHUNK_TARGET_SIZE = 30000  # 原来 110000，改为 30000
```

**效果**:
- 更多分块（更频繁的进度更新）
- 更快看到第一个结果
- 更好的实时感

**注意**: Demo 后记得改回 110000

## 🛠️ 故障处理

### 问题 1: 翻译太慢
**解决**:
1. 切换到 Quick Demo 文件
2. 检查网络连接
3. 确认使用 DeepSeek Free（最快）

### 问题 2: 观众看不清
**解决**:
1. 浏览器放大到 150%
2. 全屏演示模式
3. 准备备用投影

### 问题 3: 翻译失败
**解决**:
1. 准备预翻译的结果文件
2. 快速切换到备用文件
3. 继续展示结果和功能
